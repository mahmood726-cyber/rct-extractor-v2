PATTERNS THAT WORKED WELL
--------------------------
- Table format with "mean ± SD" in two columns (e.g., Kooncumchoo 2021)
- Explicit effect estimates with CI: "difference X (95% CI Y to Z)"
- Continuous outcomes more extractable than binary (5 vs 1)

PATTERNS THAT FAILED
---------------------
- Within-group change scores (Δmean ± SD) without between-group comparison
- Narrative descriptions without numeric data
- Results reported only as p-values or significance statements
- Dropout counts mentioned in flow diagrams but not in results text
- Multiple similar outcomes in same study (hard to disambiguate)

RECOMMENDATIONS
---------------
1. Pre-process source PDFs to fix common OCR errors (minus signs, ligatures)
2. Improve outcome name matching (synonyms, abbreviations)
3. Extract from tables more robustly (structured parsing)
4. Handle within-group changes (may need to compute between-group from change scores)
5. Add de-duplication logic when multiple outcomes extract same numbers
6. Flag suspicious extractions (e.g., OR with CI bounds > 50, duplicate extractions)
