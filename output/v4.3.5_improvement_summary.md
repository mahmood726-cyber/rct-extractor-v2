**Date:** 2026-02-03
**Based on:** Multi-persona review + Al-Fātiḥah design principles

---

## What Was Built

### 1. Extraction Schema (`src/core/extraction_schema.py`)
**Principle 1: "Start with intention"**

- Minimal, unambiguous schema locked early
- Every field has: `value`, `confidence`, `status`, `provenance`
- `ExtractionStatus`: CERTAIN / UNCERTAIN / MISSING / INVALID
- Complete `EffectInputs` for meta-analysis (events, Ns, means, SDs)
- `Outcome` with context (name, timepoint, is_primary)
- PDF hash for reproducibility

### 2. Extraction Validators (`src/core/extraction_validators.py`)
**Principle 4: "Accountability - catch silent wrongness"**

**Hard validators (errors):**
- N consistency: arm sums must match total
- CI bounds: lower < estimate < upper
- P-values: must be in [0, 1]
- Positive values: SD > 0, ratios > 0

**Soft validators (warnings):**
- N analyzed vs randomized check
- Missing critical fields for meta-analysis
- Outcome context (name, timepoint required)
- Arm balance check

### 3. Text Preprocessor (`src/core/text_preprocessor.py`)
**Principle 2: "Build reliable text layer before extraction"**

- Unicode normalization (fi ligatures, dash variants, quotes)
- Header/footer detection and removal
- Two-column layout detection and reordering
- Dehyphenation (hy-\nphen → hyphen)
- Section detection (abstract, methods, results)
- Line + page provenance preservation

### 4. CI Validation Sampling (`scripts/validate_recovered_cis.py`)
**Addresses biostatistician critique**

- Samples N recovered CIs for manual verification
- Extracts context from source PDFs
- Generates verification checklist with highlighted matches
- Tracks verification results

## Architecture Improvements

### Before (v4.3.3)
```
PDF → Text → Regex → Extractions (no validation)
```

### After (v4.3.5)
```
PDF → Text Preprocessor → Section Detection →
  Multi-pass Extraction → Validators →
  Confidence Scoring → Provenance Tracking
```

## Lessons Learned

1. **Proximity search is dangerous** - 500 char window too wide, matches wrong values
2. **Plausibility ≠ correctness** - CI can bracket value but belong to different effect
3. **Validation is essential** - 92.3% headline number was misleading without verification
4. **Schema-first design** - Helps avoid ad-hoc field additions
5. **Fail loudly** - Better to return UNCERTAIN than confident-wrong

---

## Conclusion

The architectural improvements provide a solid foundation for reliable extraction:
- Provenance tracking enables debugging
- Validators catch impossible values
- Text preprocessing handles layout issues

However, the **CI proximity search needs tightening** before the 92.3% number can be trusted. Estimated true CI completion after verification: **70-75%**, still above the 63.7% baseline.
